## Introducción

React continúa siendo la tecnología frontend más adoptada según encuestas recientes. En esta guía práctica descubrirás:

- Cómo iniciar proyectos con herramientas modernas
- La sintaxis actual de componentes funcionales
- Diferencias entre enfoques nuevos y obsoletos
- Optimizaciones esenciales desde el primer día

## Qué es React y por qué es relevante

<ImageOptimized src={import("../../assets/introReact/img1.webp")} alt="Imagen representando el logo de React" />

React es una librería JavaScript mantenida por Meta, diseñada para construir interfaces de usuario interactivas mediante componentes reutilizables. Su importancia radica en:

- Arquitectura flexible que permite integrar con otras librerías
- Comunidad activa con actualizaciones constantes
- Alto desempeño gracias al Virtual DOM
- Curva de aprendizaje accesible para principiantes

### Entendiendo el concepto de librería

React se enfoca específicamente en la capa de vista de tu aplicación. A diferencia de un framework completo:

- No impone arquitectura global
- Permite elegir soluciones para routing o estado
- Se puede integrar progresivamente en proyectos existentes

## Principales ventajas de React

1. **Componentización**: Divide interfaces en piezas independientes
2. **JSX**: Sintaxis intuitiva para mezclar HTML con lógica JS
3. **Ecosistema**: Miles de librerías compatibles (React Router, Redux, etc.)
4. **Comunidad**: Soluciones documentadas para casi cualquier problema
5. **Demanda laboral**: Habilidad más solicitada en ofertas frontend

## Desarrollo de componentes con Hooks

### Uso de useState y useEffect

**Componente contador básico:**

<CodeBlockComponent
  code={`
    import { useState, useEffect } from 'react';

    function Counter() {
      const [count, setCount] = useState(0);

      useEffect(() => {
        document.title = \`Contador: \${count}\`;
        return () => document.title = 'React App';
      }, [count]);

      return (
        <div>
          <p>Valor actual: {count}</p>
          <button onClick={() => setCount(c => c + 1)}>
            Incrementar
          </button>
        </div>
      );
    }
  `}
  language="javascript"
  showLineNumbers={true}
/>

**Explicación clave:**
- `useState` maneja el estado interno
- `useEffect` sincroniza efectos secundarios
- La dependencia `[count]` optimiza ejecuciones
- Función de limpieza restaura valores

## Conclusión y próximos pasos

Para dominar React en 2025:

1. Comienza con Vite para nuevos proyectos
2. Domina los Hooks esenciales (useState, useEffect)
3. Practica la creación de componentes reutilizables
4. Implementa optimizaciones desde el inicio
5. Explora el ecosistema (React Router, Context API)

**Ejercicio práctico recomendado:**
1. Crea un proyecto con Vite
2. Implementa un componente de formulario con validación
3. Agrega efectos de carga con useState y useEffect
4. Optimiza el rendimiento con memoización
